ทรัพยากรสาธิตคือเครื่องเสมือน Azure ที่กำหนดค่าไว้ล่วงหน้า สามารถติดตั้งใช้งานผ่าน Terraform ได้
แต่ละส่วนประกอบเป็น VM แบบสแตนด์อโลนที่มีซอฟต์แวร์เฉพาะทางติดตั้งและกำหนดค่า
ผ่าน cloud-init ในการบูตครั้งแรก

## การเลือกส่วนประกอบ

ส่วนประกอบเป็นหน่วยพื้นฐาน เลือกตามสิ่งที่การสาธิตของคุณต้องการ:

- **เซิร์ฟเวอร์ต้นทาง** ให้บริการเว็บแอปพลิเคชันที่มีช่องโหว่ซึ่งทำหน้าที่เป็นเป้าหมายสำหรับ
  ฟีเจอร์ด้านความปลอดภัยของ F5 XC การสาธิตส่วนใหญ่ต้องการส่วนนี้
- **ตัวสร้างทราฟฟิก** สร้างทราฟฟิกจากการโจมตีและทราฟฟิกปกติไปยังสภาพแวดล้อม
  สาธิตของคุณ จำเป็นเมื่อสาธิตประสิทธิภาพของ WAF การตรวจจับบอท หรือการป้องกัน API
- **ตัวจำลอง CDN** วาง CDN edge ที่ใช้ NGINX ระหว่างไคลเอนต์และเซิร์ฟเวอร์ต้นทาง
  โดยแทรกเฮดเดอร์เฉพาะผู้ให้บริการ ใช้สำหรับการสาธิตการผสานรวม CDN

## ข้อกำหนดเบื้องต้นที่ใช้ร่วมกัน

ส่วนประกอบทั้งหมดต้องการ:

- Azure subscription ที่มีสิทธิ์ระดับ contributor
- Terraform 1.5 หรือใหม่กว่า
- Azure CLI ที่ยืนยันตัวตนแล้ว (`az login`)
- คู่คีย์ SSH (เส้นทางเริ่มต้น `~/.ssh/id_ed25519.pub`)

## รูปแบบการติดตั้งใช้งาน

ทุกส่วนประกอบใช้ขั้นตอนการทำงาน Terraform เดียวกัน:

1. โคลนรีโปของส่วนประกอบ
2. `cd terraform/`
3. คัดลอก `terraform.tfvars.example` เป็น `terraform.tfvars` และตั้งค่าตัวแปรที่จำเป็น (`subscription_id` จำเป็นสำหรับทุกส่วนประกอบ บางส่วนประกอบมีอินพุตที่จำเป็นเพิ่มเติม)
4. `terraform init && terraform plan && terraform apply`
5. ตรวจสอบผ่าน health check endpoint ของส่วนประกอบ

ตัวระบุผู้ติดตั้งใช้งานจะถูกแก้ไขอัตโนมัติจากบัญชี Azure AD ของคุณ และฝังอยู่ในชื่อทรัพยากรทั้งหมด (เช่น `rg-origin-server-lab-rmordasiewicz`)

## การผสานรวมข้ามส่วนประกอบ

ส่วนประกอบต่าง ๆ รวมกันเพื่อสร้างสภาพแวดล้อมสาธิตที่สมบูรณ์:

```
Traffic Generator ──► F5 XC HTTP LB (WAF/Bot/API/CSD) ──► Origin Server
                                     ▲
                               CDN Simulator
```

**ลำดับการติดตั้งใช้งาน:**

1. **เซิร์ฟเวอร์ต้นทาง** — ติดตั้งใช้งานก่อน เนื่องจากโฮสต์แอปพลิเคชันที่ส่วนประกอบอื่น ๆ ทั้งหมดใช้เป็นเป้าหมาย
2. **F5 XC HTTP load balancer** — กำหนดค่า origin pool ที่ชี้ไปยัง public IP ของเซิร์ฟเวอร์ต้นทาง จากนั้นสร้าง HTTP load balancer พร้อมนโยบาย WAF, Bot Defense, API Security หรือ Client-Side Defense
3. **ตัวสร้างทราฟฟิก** และ/หรือ **ตัวจำลอง CDN** — ติดตั้งใช้งานเป็นลำดับสุดท้าย โดยชี้ไปยัง FQDN ของ F5 XC load balancer

ตัวแปร terraform ของแต่ละส่วนประกอบอ้างอิงผลลัพธ์ของส่วนประกอบที่ติดตั้งใช้งานไว้ก่อนหน้า ดูเอกสารของส่วนประกอบสำหรับข้อกำหนดตัวแปรเฉพาะ

ใช้ [เครื่องคำนวณราคา Azure](https://azure.microsoft.com/en-us/pricing/calculator/) สำหรับการประมาณค่าใช้จ่ายปัจจุบัน รันคำสั่ง `terraform destroy` เมื่อไม่ได้ใช้งานแล็บเพื่อหยุดค่าใช้จ่าย
