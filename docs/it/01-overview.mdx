Le risorse demo sono macchine virtuali Azure preconfigurate distribuibili tramite Terraform.
Ogni componente è una VM autonoma con software dedicato installato e configurato
tramite cloud-init al primo avvio.

## Selezione dei Componenti

I componenti sono blocchi costitutivi. Selezionateli in base alle esigenze della vostra demo:

- Il **server di origine** fornisce applicazioni web vulnerabili che fungono da obiettivi per
  le funzionalità di sicurezza di F5 XC. La maggior parte delle demo ne ha bisogno.
- Il **generatore di traffico** produce traffico di attacco e traffico legittimo contro il vostro
  ambiente demo. Necessario quando si dimostrano l'efficacia del WAF, il rilevamento dei bot o la
  protezione delle API.
- Il **simulatore CDN** posiziona un edge CDN basato su NGINX tra i client e il server di origine,
  iniettando header specifici del vendor. Utilizzato per le demo di integrazione CDN.

## Prerequisiti Condivisi

Tutti i componenti richiedono:

- Una sottoscrizione Azure con accesso contributor
- Terraform 1.5 o successivo
- Azure CLI autenticato (`az login`)
- Una coppia di chiavi SSH (percorso predefinito `~/.ssh/id_ed25519.pub`)

## Schema di Deployment

Ogni componente segue lo stesso flusso di lavoro Terraform:

1. Clonare il repository del componente
2. `cd terraform/`
3. Copiare `terraform.tfvars.example` in `terraform.tfvars` e impostare le variabili richieste (`subscription_id` è obbligatorio per tutti; alcuni componenti hanno input aggiuntivi obbligatori)
4. `terraform init && terraform plan && terraform apply`
5. Verificare tramite l'endpoint di health check del componente

Il vostro identificativo di deployer viene risolto automaticamente dal vostro account Azure AD e incorporato in tutti i nomi delle risorse (ad esempio, `rg-origin-server-lab-rmordasiewicz`).

## Integrazione tra Componenti

I componenti si combinano per creare ambienti demo completi:

```
Traffic Generator ──► F5 XC HTTP LB (WAF/Bot/API/CSD) ──► Origin Server
                                     ▲
                               CDN Simulator
```

**Ordine di deployment:**

1. **Server di origine** — distribuire per primo, ospita le applicazioni che tutti gli altri componenti prendono come obiettivo
2. **F5 XC HTTP load balancer** — configurare un origin pool che punti all'IP pubblico del server di origine, quindi creare un HTTP load balancer con policy WAF, Bot Defense, API Security o Client-Side Defense
3. **Generatore di traffico** e/o **simulatore CDN** — distribuire per ultimi, puntandoli all'FQDN del load balancer F5 XC

Le variabili Terraform di ciascun componente fanno riferimento agli output dei componenti distribuiti in precedenza. Consultate la documentazione del componente per i requisiti specifici delle variabili.

Utilizzate il [calcolatore dei prezzi Azure](https://azure.microsoft.com/en-us/pricing/calculator/) per le stime dei costi aggiornate. Eseguite `terraform destroy` quando il laboratorio non è in uso per interrompere gli addebiti.
