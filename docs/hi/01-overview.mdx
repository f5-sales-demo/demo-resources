डेमो संसाधन पूर्व-कॉन्फ़िगर Azure वर्चुअल मशीनें हैं जिन्हें Terraform के माध्यम से तैनात किया जा सकता है।
प्रत्येक घटक एक स्वतंत्र VM है जिसमें विशेष-निर्मित सॉफ़्टवेयर स्थापित और कॉन्फ़िगर किया गया है
जो पहले बूट पर cloud-init के माध्यम से सेटअप होता है।

## घटकों का चयन

घटक बिल्डिंग ब्लॉक्स हैं। अपने डेमो की आवश्यकताओं के आधार पर इनका चयन करें:

- **ऑरिजिन सर्वर** कमज़ोर वेब अनुप्रयोग प्रदान करता है जो F5 XC सुरक्षा सुविधाओं के लक्ष्य के रूप में कार्य करते हैं। अधिकांश डेमो को इसकी आवश्यकता होती है।
- **ट्रैफ़िक जनरेटर** आपके डेमो वातावरण के विरुद्ध आक्रमण और वैध ट्रैफ़िक उत्पन्न करता है। WAF प्रभावकारिता, बॉट डिटेक्शन, या API सुरक्षा प्रदर्शित करते समय इसकी आवश्यकता होती है।
- **CDN सिम्युलेटर** क्लाइंट और ऑरिजिन के बीच एक NGINX-आधारित CDN एज रखता है, जो वेंडर-विशिष्ट हेडर इंजेक्ट करता है। CDN एकीकरण डेमो के लिए उपयोग किया जाता है।

## साझा पूर्वापेक्षाएँ

सभी घटकों के लिए आवश्यक है:

- कंट्रीब्यूटर एक्सेस के साथ एक Azure सब्सक्रिप्शन
- Terraform 1.5 या बाद का संस्करण
- Azure CLI प्रमाणित (`az login`)
- एक SSH कुंजी जोड़ी (डिफ़ॉल्ट पथ `~/.ssh/id_ed25519.pub`)

## तैनाती पैटर्न

प्रत्येक घटक समान Terraform वर्कफ़्लो का पालन करता है:

1. घटक रिपो क्लोन करें
2. `cd terraform/`
3. `terraform.tfvars.example` को `terraform.tfvars` में कॉपी करें और आवश्यक वेरिएबल सेट करें (सभी के लिए `subscription_id` आवश्यक है; कुछ घटकों में अतिरिक्त आवश्यक इनपुट होते हैं)
4. `terraform init && terraform plan && terraform apply`
5. घटक के हेल्थ चेक एंडपॉइंट के माध्यम से सत्यापित करें

आपका डिप्लॉयर पहचानकर्ता आपके Azure AD खाते से स्वचालित रूप से निर्धारित होता है और सभी संसाधन नामों में एम्बेड किया जाता है (उदा., `rg-origin-server-lab-rmordasiewicz`)।

## क्रॉस-घटक एकीकरण

घटक मिलकर पूर्ण डेमो वातावरण बनाते हैं:

```
Traffic Generator ──► F5 XC HTTP LB (WAF/Bot/API/CSD) ──► Origin Server
                                     ▲
                               CDN Simulator
```

**तैनाती क्रम:**

1. **ऑरिजिन सर्वर** — पहले तैनात करें, यह उन अनुप्रयोगों को होस्ट करता है जिन्हें अन्य सभी घटक लक्षित करते हैं
2. **F5 XC HTTP लोड बैलेंसर** — ऑरिजिन सर्वर के सार्वजनिक IP की ओर इंगित करने वाला एक ऑरिजिन पूल कॉन्फ़िगर करें, फिर WAF, Bot Defense, API Security, या Client-Side Defense नीतियों के साथ एक HTTP लोड बैलेंसर बनाएँ
3. **ट्रैफ़िक जनरेटर** और/या **CDN सिम्युलेटर** — अंत में तैनात करें, F5 XC लोड बैलेंसर FQDN की ओर इंगित करते हुए

प्रत्येक घटक के terraform वेरिएबल पहले से तैनात घटकों के आउटपुट का संदर्भ देते हैं। विशिष्ट वेरिएबल आवश्यकताओं के लिए घटक दस्तावेज़ देखें।

वर्तमान लागत अनुमान के लिए [Azure मूल्य निर्धारण कैलकुलेटर](https://azure.microsoft.com/en-us/pricing/calculator/) का उपयोग करें। शुल्क रोकने के लिए लैब निष्क्रिय होने पर `terraform destroy` चलाएँ।
